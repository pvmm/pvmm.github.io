draw_counter

void draw_counter();

Draw static screen elements related to the mine counter, i. e., a frame and mine icon.

draw_game_over

void draw_game_over(minefield* mf);

Draws a game over message when the game ends. It will be called when the game ends if the DRAW_GAME_OVER macro is defined.

draw_minefield

void draw_minefield(minefield* mf);

Draws the background of the boardgame on screen.

draw_minefield_contents

void draw_minefield_contents(minefield* mf);

Draws the playfield of the boardgame on screen.

draw_scenario

void draw_scenario(minefield* mf);

Draw the background image that surrounds the board.

ðŸ“Œ Implementation details

If coding a tile-based platform that uses tiles.h header, you can use the definition in 8x8_tiles.h or 16x16_tiles.h instead of writing your own.

Note: some tile-based platforms use the GROUND tile to cover the whole background area.

See tile_index for the index value of the GROUND tile.

draw_timer

void draw_timer();

Draw static screen elements related to the time counter, i. e., a frame and ampersand icon.

draw_title_screen

void draw_title_screen(minefield* mf);

Draws a title screen message when the game starts. It will be called at the start of the game if the DRAW_TITLE_SCREEN macro is defined.

free_minefield

void free_minefield(minefield* mf);

Free memory and resources of the board. The default destructor is called if the MINEFIELD_ALLOCATION macro is not defined.

ðŸ“Œ Implementation details

  • The same resources allocated by init_minefield() should be released by this function.

idle_update

void idle_update(minefield* mf);

Runs things in background, like music and animations.

ðŸ“Œ Implementation details

  • Refresh game state, like animation and music and when finished just call wait_ticks() to wait for the next frame.

init_minefield

minefield* init_minefield();

Set up memory and resources of the board. The default constructor is called if the MINEFIELD_ALLOCATION macro is not defined.

ðŸ“Œ Implementation details

Depending on available system resources and compiler features, you may prefer to statically reserve memory or allocate it dynamically.

platform_main_loop

void platform_main_loop(minefield* mf);

Reimplements the game loop on platforms that process their own event loop. It will be called if the MAIN_LOOP_REIMPLEMENTED macro is defined.

reset_minefield

void reset_minefield(minefield* mf);

Set minefield to initial state, setting number of bombs and their positions in the grid. A default initialisation function is called if the RESET_MINEFIELD macro is not defined.

reset_timer

void reset_timer();

Reset elapsed time before new game starts.

ðŸ“Œ Implementation details

  • If ENABLE_TIMER is not defined, this function will not be called and the timer will not be displayed and updated during the game loop.

  • The set of tile characters must include all the digits and the colon to separate minutes and seconds.

set_minefield_cell

void set_minefield_cell(minefield* mf, uint8_t x, uint8_t y, uint8_t input);

Update minefield cell directly for mouse, lightpen, light gun or any other type of direct input device. input is either MINE_INPUT_OPEN, MINE_INPUT_OPEN_BLOCK or MINE_INPUT_FLAG, while x and y are the minefield cell coordinates.

update_counter

void update_counter(minefield* mf);

Display mine counter on screen.

ðŸ“Œ Implementation details

  • If ENABLE_COUNTER is not defined, this function will not be called and the mine counter will not be displayed and updated during the game loop.

  • The set of tile characters must include all the digits and the negative sign, since the counter may become negative.
